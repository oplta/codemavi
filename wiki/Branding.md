# Branding Guidelines

- Name usage: Code Mavi IDE (never repeat as "Code Mavi IDE IDE").
- Repo references: use codemavi-ide for URL fragments and packaging references when appropriate.
- Color palette: base #020b18; accent #3b82f6. Use these consistently across UI and docs.
- Logo usage: maintain clear, high-contrast logos; avoid stretched or recolored logos outside approved variants.
- Language consistency: refer to the product as Code Mavi IDE in all docs; avoid mixed branding phrases.
- Example branding statements:
  - Correct: Code Mavi IDE is an open-source agent-first IDE.
  - Incorrect: Code Mavi IDE IDE project.
